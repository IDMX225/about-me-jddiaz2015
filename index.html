Image Hover Effects with HTML & CSS

Hover effects are a powerful way to enhance user experience on websites. When a user hovers their mouse over an image or element, the site responds visually — which makes it feel interactive and modern...

This image uses a combination of transitions, transforms, and opacity to display a caption smoothly...

How can we make this effect on HTML and CSS?

The <figure id="hover-demo"> element wraps the image and its caption. The <img> tag shows the image, and the <figcaption> with a <h4> contains the text that appears on hover. CSS makes the caption fade in over the image smoothly.

The #hover-demo styles the <figure> container by setting it to position: relative for internal absolute positioning, overflow: hidden to contain any zoom effects, max-width: 400px and width: 100% for responsive sizing, margin: 2rem auto to center it with vertical spacing, and border-radius: 10px to round its corners.

The #hover-demo img styles the image by setting it to display: block, making it responsive with width: 100% and height: auto, removing border-radius, margin, and padding for a clean appearance.

The #hover-demo figcaption is positioned absolutely within the <figure> element, covering the entire image with width: 100% and height: 100%. The background: rgba(0, 0, 0, 0.5) creates a dark overlay, and the content is centered both horizontally and vertically using display: flex, justify-content: center, and align-items: center. The caption starts invisible with opacity: 0 and fades in on hover, thanks to transition: opacity 0.8s ease. Padding, margin, and border-radius are set to 0 to remove extra space and make the caption fit perfectly over the image, while pointer-events: none ensures that the caption doesn't interfere with interactions on the image.

The #hover-demo:hover img rule zooms in the image by scaling it to 1.05 times its original size when the user hovers over the <figure> element. The #hover-demo:hover figcaption rule makes the caption visible with a smooth fade-in effect by setting its opacity to 1 during hover. Additionally, the #hover-demo figcaption h4 rule changes the text color of the <h4> inside the <figcaption> to white, ensuring good contrast against the dark overlay background.

Animated Moving Dot

HOW DOES THE DOT MOVE?

The <section id="moving-dot-demo"> groups the animated dot and its title so they can be styled together. Inside it, the <h2> adds a heading that labels the effect, and the <div class="moving-dot"> creates the dot itself. This dot is connected to CSS animations through its class name, which lets it move smoothly across the screen without any interaction like hovering.

The #moving-dot-demo selector styles the entire section where the animated dot appears. It sets a soft background color with background-color: #fef6e4 to make the animation stand out, centers all content using text-align: center, and adds generous spacing around the content with padding: 3rem for a clean, balanced layout.

The .moving-dot class styles the animated dot by giving it a width and height of 20px to form a small circle. The background-color: #8bd3dd gives it a soft teal color, and border-radius: 50% makes the shape perfectly round. The dot is centered on the page using margin: 2rem auto, and the position: relative allows it to move within its container. Finally, the animation: moveDot 2s infinite ease-in-out applies a smooth back-and-forth motion using a keyframe animation named moveDot that loops endlessly.

The @keyframes moveDot rule defines the animation for the moving dot. At 0%, the dot is positioned at the leftmost point (left: 0). At 50%, the dot moves 100px to the right (left: 100px), and by 100%, it returns to its original position (left: 0). The animation moves the dot back and forth horizontally, creating a smooth motion that loops infinitely.
